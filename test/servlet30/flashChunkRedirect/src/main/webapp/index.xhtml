Facelet Title This application adds a value to the flash in an Ajax ActionListener. The Ajax response is large enough to be chunked. Therefore the JSF1095 warning is displayed and the flash cookie not correctly written. The same thing happens if the value is added in a not in a normal request and navigation is done to a page that is large enough to be chunked. See second link. If the session is cleared and page2 is directly called (/faces/page2.xhtml) an exception will occur:
java.lang.IllegalStateException: PWC3999: Cannot create a session after the response has been committed

It seems to be the same problem with the chunked response. As page2 is large enough to be chunked and contains a form an exception seems to occur when the session for Server-Side State Saving is created.
